## 配置说明

### 层级配置示例

```javascript
// 小规模测试（推荐首次运行）
const subordinates = [2, 2, 2];  // 总共 2+2+2 = 6人

// 中等规模
const subordinates = [3, 5, 10]; // 总共 3+5+10 = 18人

// 大规模测试
const subordinates = [5, 10, 20]; // 总共 5+10+20 = 35人
```

### 充值配置

充值金额范围：5000-15000（随机）  
打码量倍数：1倍  

如需修改，编辑 `inviteService.js` 中的：
```javascript
const rechargeAmount = Math.floor(Math.random() * (15000 - 5000 + 1)) + 5000;
```

## 常见问题

### Q: 验证码错误怎么办？
A: 系统已优化验证码获取逻辑，通过账号查询最新验证码。如果还有问题，检查：
- 管理员token是否有效
- 验证码是否过期（等待时间太长）

### Q: 注册失败怎么办？
A: 系统会自动切换：
1. 先尝试手机号注册
2. 失败则自动尝试邮箱注册
3. 都失败则停止流程并报错

### Q: 充值失败会影响其他用户吗？
A: 不会。单个用户充值失败只会跳过该用户的投注，继续处理下一个用户。

### Q: 如何查看详细日志？
A: 所有操作都有详细的控制台输出，包括：
- 每个用户的注册状态
- userId 和 inviteCode
- 充值和投注结果
- 最终统计报告

## 测试建议

1. **首次运行**: 使用小规模配置 `[2, 2, 2]`
2. **验证成功**: 检查统计报告，确认流程正常
3. **扩大规模**: 逐步增加人数 `[3, 5, 10]`
4. **生产环境**: 根据需要配置更大规模
