# 批量充值提现测试工具

## 📖 简介

这个工具提供了两种批量测试方案：
1. **批量充值提现**：注册 → 充值 → 提现
2. **批量充值投注提现**：注册 → 充值 → 投注 → 提现

## 📁 文件说明

| 文件 | 说明 |
|------|------|
| `batchRechargeWithdraw.test.js` | 批量充值提现测试（不含投注） |
| `batchRechargeWithdrawWithBet.test.js` | 批量充值投注提现测试（含投注） |
| `run-batch-recharge-withdraw.sh` | 充值提现测试执行脚本 |
| `run-batch-with-bet.sh` | 充值投注提现测试执行脚本 |

## 🚀 快速开始

### 方案1: 批量充值提现（不含投注）

```bash
# 基础用法：10个账号
./run-batch-recharge-withdraw.sh 3004

# 指定账号数量：20个账号
./run-batch-recharge-withdraw.sh 3004 20

# 自定义充值金额范围
./run-batch-recharge-withdraw.sh 3004 10 --min-recharge=2000 --max-recharge=10000

# 启用后台审核
./run-batch-recharge-withdraw.sh 3004 10 --enable-approval
```

### 方案2: 批量充值投注提现（含投注）

```bash
# 基础用法：10个账号，每个投注5轮
./run-batch-with-bet.sh 3004

# 指定账号数量和投注轮数
./run-batch-with-bet.sh 3004 20 --bet-rounds=10

# 自定义充值金额和投注轮数
./run-batch-with-bet.sh 3004 10 --min-recharge=2000 --max-recharge=10000 --bet-rounds=8

# 完整配置
./run-batch-with-bet.sh 3004 2 --min-recharge=2000 --max-recharge=8000 --bet-rounds=1 --withdraw-ratio=0.3 --enable-approval
```

## 📋 环境变量说明

### 通用参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `TENANT_ID` | 租户ID | 必需 |
| `ACCOUNT_COUNT` | 账号数量 | 10 |
| `MIN_RECHARGE` | 最小充值金额 | 1000 |
| `MAX_RECHARGE` | 最大充值金额 | 5000 |
| `WITHDRAW_RATIO` | 提现比例（0-1） | 0.8 |
| `ENABLE_BACKEND_APPROVAL` | 是否启用后台审核 | false |
| `CONCURRENT_SIZE` | 并发处理数量 | 5 |

### 投注专用参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `BET_ROUNDS` | 每个账号投注轮数 | 5 |

## 📊 测试流程

### 方案1: 充值提现流程

```
1. 批量注册账号
   ├─ 优先手机号注册
   └─ 失败则邮箱注册

2. 批量充值
   ├─ 优先前台充值
   └─ 失败则后台充值兜底

3. 添加提现钱包
   └─ 添加所有类型钱包

4. 批量提现
   ├─ 检查余额
   ├─ 检查提现条件
   ├─ 选择提现通道
   └─ 提交提现申请

5. 后台审核（可选）
   └─ 自动审核通过
```

### 方案2: 充值投注提现流程

```
1. 批量注册账号
   ├─ 优先手机号注册
   └─ 失败则邮箱注册

2. 批量充值
   ├─ 优先前台充值
   └─ 失败则后台充值兜底

3. 批量投注
   ├─ 获取投注token
   ├─ 获取期号
   ├─ 执行投注
   └─ 完成打码量

4. 添加提现钱包
   └─ 添加所有类型钱包

5. 批量提现
   ├─ 检查余额
   ├─ 检查打码量
   ├─ 选择提现通道
   └─ 提交提现申请

6. 后台审核（可选）
   └─ 自动审核通过
```

## ⚠️ 注意事项

1. **打码量要求**
   - 如果平台有打码量要求，必须使用带投注的测试
   - 投注轮数需要根据打码量要求调整
   - 打码量未完成时无法提现

2. **充值方式**
   - 优先使用前台充值（真实模拟用户行为）
   - 前台充值失败时自动切换到后台充值兜底
   - 后台充值需要管理员权限

3. **提现通道**
   - 自动排除 UPI 通道
   - 随机选择可用通道
   - 需要提前添加钱包信息

4. **并发控制**
   - 默认并发数为5，避免请求过快
   - 可通过 `--concurrent` 参数调整
   - 建议根据服务器性能调整

5. **后台审核**
   - 默认不启用后台审核
   - 启用后会自动审核通过所有提现
   - 需要管理员权限

## 🔍 故障排查

### 问题1: 充值失败

**可能原因**：
- 充值通道不可用
- 金额超出限制
- 网络问题

**解决方案**：
- 检查充值通道配置
- 调整充值金额范围
- 查看日志详细信息

### 问题2: 投注失败

**可能原因**：
- 余额不足
- 期号不可投注
- Token过期

**解决方案**：
- 确保充值成功
- 等待下一期开始
- 检查token获取逻辑

### 问题3: 提现失败

**可能原因**：
- 打码量未完成
- 提现次数用完
- 余额不足

**解决方案**：
- 增加投注轮数
- 检查提现配置
- 查看余额信息

### 问题4: 注册失败

**可能原因**：
- 手机号已存在
- 邮箱已存在
- 验证码问题

**解决方案**：
- 工具会自动切换手机号/邮箱
- 检查验证码服务
- 查看注册日志
