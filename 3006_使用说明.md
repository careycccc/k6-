# 3006租户多语言活动创建 - 使用说明

## ⚠️ 重要提示

**必须通过 `-e LANGUAGES=bn,en,zh` 参数指定语言，否则会使用默认语言(zh,en,hi)，不包含孟加拉语！**

## 正确的使用方法

### 方法1：使用便捷脚本（推荐）✅

这些脚本已经包含了正确的语言参数：

```bash
cd k6-

# 创建优惠券
./run_3006_coupon.sh

# 创建系统活动
./run_3006_system_active.sh

# 创建所有活动
./run_3006_all_activities.sh
```

### 方法2：直接使用k6命令 ✅

**必须包含 `-e LANGUAGES=bn,en,zh` 参数：**

```bash
cd k6-/k6

# 创建优惠券（正确）
k6 run -e LANGUAGES=bn,en,zh -e ACTIVITIES=coupon tests/api/script/testActive_3006_multilang.js

# 创建签到（正确）
k6 run -e LANGUAGES=bn,en,zh -e ACTIVITIES=signin tests/api/script/testActive_3006_multilang.js

# 创建多个活动（正确）
k6 run -e LANGUAGES=bn,en,zh -e ACTIVITIES=coupon,signin,banner tests/api/script/testActive_3006_multilang.js
```

## ❌ 错误的使用方法

```bash
# 错误：没有指定LANGUAGES参数，会使用默认语言(zh,en,hi)，不包含孟加拉语
k6 run -e ACTIVITIES=coupon tests/api/script/testActive_3006_multilang.js

# 错误：没有指定LANGUAGES参数
k6 run tests/api/script/testActive_3006_multilang.js
```

## 如何验证语言配置

运行脚本后，查看日志中的语言配置信息：

```
========== 语言配置 ==========
激活语言数: 3
  bn - 孟加拉语
  en - 英语
  zh - 中文
==============================
```

如果看到的是：
```
激活语言数: 3
  zh - 中文
  en - 英语
  hi - 印地语
```

说明没有正确指定LANGUAGES参数！

## 常见问题

### Q: 为什么创建的活动没有孟加拉语？

A: 因为没有通过 `-e LANGUAGES=bn,en,zh` 指定语言。请使用便捷脚本或确保命令中包含此参数。

### Q: 如何查看当前使用的语言？

A: 运行脚本后，查看日志中的"语言配置"部分，会显示激活的语言列表。

### Q: 可以只创建某一种语言吗？

A: 可以，例如只创建孟加拉语：`-e LANGUAGES=bn`

### Q: 可以添加其他语言吗？

A: 可以，支持的语言包括：zh(中文), en(英语), hi(印地语), es(西班牙语), pt(葡萄牙语), vi(越南语), ur(乌尔都语), ms(马来语), bn(孟加拉语)

## 技术说明

语言配置通过 `k6/config/languageConfig.js` 中的 `getActiveLangs()` 函数读取环境变量 `LANGUAGES`。

如果不指定 `LANGUAGES` 环境变量，会使用默认语言：zh,en,hi（中文、英语、印地语）。

因此，**必须显式指定 `-e LANGUAGES=bn,en,zh` 才能包含孟加拉语**。
