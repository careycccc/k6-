# 3001租户 - 站内信和活动资讯创建说明

## 问题排查

### 1. 活动资讯（activityGuide）

**问题**: 只创建了英文版本，中文和印地语没有创建

**原因**: 活动资讯不是多语言活动，它是系统级的配置，不需要为每种语言单独创建。

**说明**:
- 活动资讯是引导用户观看充值教程视频的功能
- 它只有一个配置，对所有语言的用户都生效
- 配置内容包括：视频URL、观看时长、奖励金额等
- 这是正常行为，不是bug

### 2. 站内信（inmail）

**问题**: 一个都没有创建

**可能原因**:
1. 图片上传失败
2. 优惠券查询失败（站内信可以附带优惠券）
3. API调用失败

**解决方案**: 已添加详细日志，运行脚本时会显示具体失败原因

## 使用脚本

### 方式1：只创建站内信

```bash
./run_3001_inmail.sh
```

这会为3001租户创建站内信，支持三种语言：
- 中文（zh）
- 英语（en）
- 印地语（hi）

### 方式2：只创建活动资讯

```bash
./run_3001_activityGuide.sh
```

这会更新活动资讯配置（系统级，不区分语言）

### 方式3：同时创建两个活动

```bash
./run_3001_inmail_and_guide.sh
```

这会同时创建站内信和活动资讯

### 方式4：自定义语言

```bash
# 只使用中文和英语创建站内信
LANGUAGES=zh,en ./run_3001_inmail.sh

# 使用所有三种语言（默认）
LANGUAGES=zh,en,hi ./run_3001_inmail.sh
```

## 站内信多语言支持

站内信会为每种激活的语言创建对应的翻译版本：

### 中文（zh）
- 标题：`[跳转类型] 站内信`
- 内容：包含图片和中文描述

### 英语（en）
- 标题：`[Jump Type] In-Mail`
- 内容：包含图片和英文描述

### 印地语（hi）
- 标题：`[Jump Type] इन-मेल`
- 内容：包含图片和印地语描述

## 站内信创建流程

1. **上传图片资源**
   - 自动上传站内信所需的图片
   - 如果已有图片资源则跳过上传

2. **查询优惠券**（可选）
   - 尝试查询系统中的优惠券
   - 如果有优惠券，会附加到站内信中
   - 如果没有优惠券，站内信仍然可以创建

3. **创建站内信**
   - 根据 jumpType 创建多条站内信
   - 每条站内信包含多语言翻译
   - 每次创建间隔0.5秒，避免频率限制

4. **启用站内信**
   - 自动启用新创建的站内信
   - 每次启用间隔1秒

## 活动资讯配置说明

活动资讯是系统级配置，包含以下内容：

- **活动名称**: 充值视频观看
- **活动类型**: 视频观看引导
- **视频URL**: 充值教程视频地址
- **观看时长**: 10秒（用户需要观看的最少时长）
- **奖励金额**: 25（观看后发放的奖励）
- **打码倍数**: 1倍

## 常见问题

### Q1: 站内信创建失败，提示图片上传失败

**A**: 检查图片文件是否存在：
```bash
ls -la k6/tests/api/uploadFile/img/inmail/
```

如果图片不存在，需要准备图片文件。

### Q2: 站内信创建成功但没有优惠券

**A**: 这是正常的，站内信可以不附带优惠券。如果需要优惠券：
1. 先创建优惠券活动
2. 再创建站内信

### Q3: 活动资讯只有一个配置，是否正常？

**A**: 是的，这是正常的。活动资讯是系统级配置，不区分语言。

### Q4: 如何确认站内信的多语言版本都创建了？

**A**: 查看日志输出，会显示：
```
[createInmail] 当前激活的语言: zh, en, hi
[createInmail] 将为每种语言创建站内信翻译
```

每条站内信的 `translations` 字段会包含所有语言的版本。
